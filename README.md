# 航运管理系统



## 技术栈

SpringBoot2 + Vue3 + MybatisPlus + MySQL + Lombok



**管理员**关键功能涵盖个人中心、船员管理、航行安全管理、船舶信息管理、船舶调度管理、航运货物管理、航运安全监控管理、系统管理



航行安全管理模块用于监控和管理船舶航行安全相关事务；船舶信息管理模块负责船舶基本信息的维护；船舶调度管理模块制定和调整船舶调度计划；航运货物管理模块跟踪货物运输状态；航运安全监控管理模块实时监测船舶安全状况，船舶调度模块，承担船舶调度计划的制定、调整以及调度任务的分配，可以根据发出地和终点智能推荐线路，确定出发港和终点港，并且显示距离，智能航线推荐这是一个按钮，点击之后才会进行推荐让用户选择，用户也可以自己手动选择线路，智能航线推荐使用最短路径算法



**船员**关键功能包括个人中心、航行安全管理、船舶调度管理、航运货物管理、航运安全监控管理

船员通过个人中心查看和修改个人信息；在航行安全管理模块反馈航行安全问题；船舶调度管理模块接收调度任务；航运货物管理模块了解货物运输情况；航运安全监控管理模块协助进行安全监控工作



航运管理系统主要包含以下功能模块：用户管理模块，负责管理员与船员的信息管理，涵盖注册、登录、信息修改等功能；船舶调度模块，承担船舶调度计划的制定、调整以及调度任务的分配；航运货物管理模块，实现对货物的运输信息跟踪、货物装卸管理等；航行安全管理模块，聚焦船舶航行安全相关事务，包括安全风险评估、安全措施制定等；船舶信息管理模块，对船舶的基本信息，如船名、船型、尺寸、载重能力等进行维护；航运安全监控模块，借助技术手段实时监测船舶的航行状态、周边危险区域等；系统管理模块，用于系统的基础配置、权限管理、日志记录等操作



以下是部分表：

船舶调度：创建时间、船名、船型、尺寸、调度安排、装载顺序、分配航线、调度详情、调度日期、用户名、经度、纬度、地址

港口表：name、latitude、longitude

航线节点关系：港口表id等

用户表：username、password、image、role(默认管理员)、create_time

token表：userid、username、table_name、role、token、create_time、expiratedtime

系统日志表：username、operation、method、params、ip、create_time

航运货物：船名、始发地、目的地、预装时间、货物种类、货物吨数、货物体积、运输价格、运输状态、用户名、日期、地址、创建时间

航运安全监控：船名、船型、航行速度、航向、附近危险区、预警措施、安全风险、时间、用户名、地址、创建时间

航行安全：船只名称、航行状态、航运安全、发布时间、创建时间

船员：用户名、密码、用户电话、用户邮箱、状态、创建时间

船舶信息：船名、船型、尺寸、载重能力、识别码、持证情况、状态、停靠地址
